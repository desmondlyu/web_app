# TTO Analysis — NOR Flash CP Test 時間優化分析工具

一個純靜態網頁版的 CP 測試時間分析平台，直接用瀏覽器開啟即可使用。

**作者**：PP32 YPLU (Desmond Lyu)

---

## 📁 支援的上傳檔案

### 必填檔案

| 檔案類型 | 命名規則 | 說明 |
|----------|---------|------|
| **CP Summary Excel** | `產品別_CP_Summary.xlsx` | 各站點的良率資料及 CAT 失效統計 |
| **MSS Excel** | `產品別_CP_MSS.xlsx` | 測試項目定義及測試站點分配 |

### 選填檔案

| 檔案類型 | 命名規則 | 說明 |
|----------|---------|------|
| **Rawdata TXT** | `產品別_站點*.txt` | 測試執行頻率與實際執行時間（可多選） |
| **TTLOG** | `產品別_Master_Summary.xlsx` | 測試時間總結資料（Master_Summary 分頁） |

---

## 📋 檔案命名規則

### CP Summary
- **格式**：`產品別_CP_Summary.xlsx`
- **範例**：`FAG112_CP_Summary.xlsx`, `EAG119C_CP_Summary.xlsx`

### MSS
- **格式**：`產品別_CP_MSS.xlsx`
- **範例**：`FAG112_CP_MSS.xlsx`, `EAG119C_CP_MSS.xlsx`

### Rawdata TXT
- **格式**：`產品別_站點*.txt`
- **範例**：`FAG112_DS00.txt` 或 `FAG112_DS00_DATALOG_X37Y16_DBIN1.txt`
- **允許的產品別前綴**：FAG*, EAG*, MAG*, AAG*, FCG*, FBG*
- **允許的站點**：DS00, S1P1, DS03, DS05, SFIN, SPRE, DS07, DS08, DS09, DS04
- **多檔案規則**：同時上傳多個 Rawdata 時，所有檔案必須來自同一產品別

### TTLOG
- **格式**：`產品別_Master_Summary.xlsx`
- **說明**：必須包含 Master_Summary 工作表，含測試項目、各站點時間統計
- **限制**：最多上傳一個檔案；若同時上傳 Rawdata，將自動清除此檔案

## 🚀 使用流程

### 步驟一：上傳分析檔案

1. **CP Summary** — 點擊或拖曳上傳（必填）
2. **MSS** — 點擊或拖曳上傳（必填）
3. **Rawdata TXT 或 TTLOG** — 至少上傳其中一個（選填）
   - 不能同時上傳 Rawdata 和 TTLOG；若上傳其中一個，系統會自動清除另一個並顯示提示「你只能用 RAWDATA 或是 TTLOG 進行分析」

### 步驟二：設定分析閾值

- **🟢 極低風險閾值**（預設 50 ppm）：低於此值判定為「低風險可移除」
- **🟡 低風險閾值**（預設 500 ppm）：低於此值判定為「建議觀察」

### 步驟三：執行分析

- 上傳 CP Summary + MSS 後，「開始分析」按鈕將啟用
- 點擊「開始分析」開始分析流程

## 📊 儀表板介紹

### 全局 KPI 卡片

| KPI | 說明 |
|-----|------|
| **總體測試時間** | 所有測試項目的累計執行時間（Rawdata 時為各站點加總；TTLOG 時為 Grand_Total_Time 加總） |
| **重複測試執行次數** | 測試在多個 Wafer 上重複執行的次數（需上傳 Rawdata） |
| **最耗時站點** | 測試時間最長的單個站點 |
| **MSS 總測試項目** | 所有站點的測試項目總數 |

### Top 10 最耗時 Test Items

跨站點排序，依實際執行時間排列，顯示：
- 排名、項目名稱、所屬站點
- 執行次數（Rawdata 來源時）
- 總執行時間
- 占總體測試時間的百分比

### 站點分析面板

切換不同站點時顯示以下內容：

#### 迷你 KPI
- **總項目**：該站點的測試項目總數
- **可移除**：從未失效或失效率極低的項目數
- **低風險**：失效率低但需持續監控的項目數
- **當站測試時間**：該站點所有測試項目的總時間
- **Yield Loss**：失效率高的項目數（不建議移除）

#### 狀態分佈圖
環形圖展示該站點各類測試項目的占比分布

#### 測試項目分類

| 分類 | 顏色 | 失效率條件 | 建議 |
|------|------|----------|------|
| 從未失效 | 🟢 | avg PPM = 0 | 可考慮移除 |
| 低風險可移除 | 🔵 | avg PPM < 50 ppm | 低風險，可評估移除 |
| 建議觀察 | 🟡 | avg PPM < 500 ppm | 建議持續監控 |
| Repair 保護 | 🟣 | test_item 含 "repair" | 即使失效率 0 也應保留 |
| 有 Yield Loss | 🔴 | avg PPM ≥ 500 ppm | 不建議移除 |

#### 三張詳細表格

1. **✂ 可移除項目表**
   - 從未失效或失效率極低的項目
   - 含失效率、CAT 代碼、執行時間等資料
   - 可排序、搜尋

2. **🔧 Repair 保護項表**
   - 含 "repair" 關鍵字的測試項目
   - 即使失效率為 0 也應保留

3. **⛔ 不建議移除表**
   - 失效率 ≥ 500 ppm 的項目
   - 標記為「有 Yield Loss」

### 可節省時間

每個站點下方顯示「**可節省時間**」，計算方式為該站點所有**可移除項目**的總測試時間。此數值不會超過「當站測試時間」。

## 📤 匯出報告

點擊右上角 **📤 匯出 Excel** 可下載完整分析報告，包含以下工作表：

1. **KPI_總覽** — 各站點統計摘要（可移除項目數、Repair 保護項、失效項目等）
2. **CP_可移除** — 所有可移除測試項目詳細列表
3. **CP_不建議移除** — 有 Yield Loss 的項目詳細列表
4. **CP_Repair保護** — Repair 保護項詳細列表
5. **Rawdata_執行** — 測試執行頻率統計（有上傳 Rawdata 時才有此工作表）

## ❓ 常見問題

**Q：為什麼我的檔案上傳被拒絕了？**  
A：檔案名稱格式不符合規則。請確保名稱正確，例如：
- CP Summary：`FAG112_CP_Summary.xlsx`
- MSS：`FAG112_CP_MSS.xlsx`
- Rawdata：`FAG112_DS00.txt`

**Q：Rawdata 和 TTLOG 可以同時上傳嗎？**  
A：不可以。若同時上傳，系統會提示並自動清除其中一個。

**Q：分析結果顯示 0 ppm 是什麼意思？**  
A：表示該站點在所有 wafer 上沒有失效，是正常現象。

**Q：「當站測試時間」和「可節省時間」的關係是什麼？**  
A：「可節省時間」必定 ≤ 「當站測試時間」，因為它只計算可移除項目的時間。
