# TTO Analysis Web App

NOR Flash CP Test 時間優化分析工具 — 純靜態網頁版，直接用瀏覽器開啟即可使用。

**✨ 新增功能**：支援新格式 CP Summary Excel（單頁 + Process 欄）；舊格式自動相容！

## 📁 支援的輸入檔案

| 檔案類型 | 命名規則 | 說明 |
|----------|---------|------|
| **CP Summary Excel** | `產品別_CP_Summary.xlsx` | 各站點良率資料（必填）|
| **MSS Excel** | `產品別_CP_MSS.xlsx` | 測試項目定義（必填）|
| **Rawdata TXT** | `產品別_站點_DATALOG_*.TXT` | 執行頻率 + 時間資料（選填，可多選）|

### 檔案命名規則

**CP Summary** 範例：`EAG119C_CP_Summary.xlsx`
- 格式：`產品別_CP_Summary.xlsx`

**MSS** 範例：`EAG119C_CP_MSS.xlsx`
- 格式：`產品別_CP_MSS.xlsx`

**Rawdata TXT** 範例：`EAG119C_DS00_DATALOG_20260425.txt`
- 格式：`產品別_站點_DATALOG_*.TXT`
- 允許的產品別前綴：**FAG*, EAG*, MAG*, AAG*, FCG*, FBG***
- 允許的站點：**DS00, S1P1, DS03, DS05, SFIN, SPRE, DS07, DS08, DS09, DS04**
- 多個 Rawdata 檔案必須來自同一產品別

## 📊 使用步驟

### 步驟一：上傳分析檔案

1. 點擊或拖曳上傳 **CP Summary Excel** 檔案（必填）
2. 點擊或拖曳上傳 **MSS Excel** 檔案（必填）
3. （可選）點擊或拖曳上傳 **Rawdata TXT** 檔案（可多選）
   - 檔案名稱格式檢查會自動進行，格式不符會被拒絕並顯示錯誤提示

### 步驟二：設定分析閾值

- **🟢 極低風險閾值**（預設 50 ppm）：低於此值判定為「低風險可移除」
- **🟡 低風險閾值**（預設 500 ppm）：低於此值判定為「建議觀察」

### 步驟三：執行分析

- 上傳 CP Summary + MSS 後，「開始分析」按鈕將啟用
- 點擊「開始分析」執行完整分析流程

## 📊 分析結果

### 全局 KPI

- **總體測試時間**：所有站點的累計執行時間
- **重複測試執行次數**：測試在多個 Wafer 上重複執行的次數
- **最耗時站點**：測試時間最長的單個站點
- **MSS 總測試項目**：所有站點的測試項目總數

### Top 10 最耗時 Test Items

依 Rawdata 實際執行時間排序（跨站點），顯示：
- 排名、項目名稱、站點
- 執行次數
- 總時間
- **佔比**：相對於總體測試時間的百分比

### 站點分析

每個站點提供：

1. **迷你 KPI 卡片**：項目統計概覽
2. **狀態分佈環形圖**：各類別測試項目占比
3. **✂ 可移除項目表**
   - 從未失效或失效率極低的項目
   - 可排序、搜尋
4. **🔧 Repair 保護項表**
   - 含 "repair" 關鍵字的測試項目
   - 即使失效率為 0 也應保留
5. **⛔ 不建議移除表**
   - 有 Yield Loss 的項目（失效率 ≥ 500 ppm）

### 測試項目分類

| 狀態 | 條件 | 建議 |
|------|------|------|
| 🟢 從未失效 | 所有 CAT 的 avg PPM = 0 | 可考慮移除 |
| 🔵 低風險可移除 | avg PPM < 50 ppm | 低風險，可評估移除 |
| 🟡 建議觀察 | avg PPM < 500 ppm | 建議持續監控 |
| 🟣 Repair 保護 | test_item 含 "repair" | 即使 0 fail 仍應保留 |
| 🔴 有 Yield Loss | avg PPM ≥ 500 ppm | 不建議移除 |

### PPM 計算

```
avg_ppm = (所有 wafer 的失效% 平均) × 10000
max_ppm = (最高失效%) × 10000
```

例：失效 0.05% → **500 ppm**

### CAT（失效代碼）規則

- **Pass CAT**（不計入失效）：`01`, `02`, `03`, `04`, `05`, `07`
- **Fail CAT**：其他所有 CAT 代碼（如 `0A`, `0F`, `1K`, `2N`, `5V` 等）

## 📤 匯出報告

點擊右上角 **📤 匯出 Excel** 可下載完整分析報告，包含 5 個工作表：

1. **KPI_總覽** — 各站點統計摘要（可移除項目數、Repair 保護項、失效項目等）
2. **CP_可移除** — 所有可移除測試項目詳細列表
3. **CP_不建議移除** — 有 Yield Loss 的項目詳細列表
4. **CP_Repair保護** — Repair 保護項詳細列表
5. **Rawdata_執行** — 測試執行頻率統計（有上傳 Rawdata 時才有此工作表）

## 🛠 本地開發

直接用瀏覽器開啟 `index.html` 即可（**不需要**本地伺服器）。

所有依賴均透過 CDN 載入：
- **Tailwind CSS 4** — UI 框架
- **Chart.js 4.4** — 環形圖表
- **SheetJS 0.20.2** — Excel 讀寫

## 📝 注意事項

- 所有分析在**瀏覽器端進行**，檔案不會上傳至任何伺服器，資料完全私密
- 大型 Rawdata TXT（>10 MB）解析可能需要 5–15 秒，請耐心等待
- 500 列以上的表格使用虛擬捲動以確保效能
- 支援**新舊兩種 CP Summary Excel 格式**，系統自動偵測：
  - **新格式**：單一工作表，含 Process 欄位用於站點分組
  - **舊格式**：多個工作表（每個 Sheet 代表一個站點）

## ❓ 常見問題

**Q：為什麼我的檔案上傳被拒絕了？**  
A：檔案名稱格式不符合規則。請確保：
- CP Summary：`產品別_CP_Summary.xlsx`
- MSS：`產品別_CP_MSS.xlsx`
- Rawdata：`產品別_站點_DATALOG_*.TXT`（產品別和站點必須符合允許清單）

**Q：為什麼分析結果為 0 ppm？**  
A：這是正常現象，表示該站點在所有 wafer 上沒有失效。

**Q：Rawdata 匯出的檔案無法開啟？**  
A：請使用 Excel 2016 或更新版本，或使用 LibreOffice Calc 開啟。

**Q：可以支援多個 Rawdata 檔案嗎？**  
A：可以。請一次上傳多個檔案，但所有檔案必須來自同一產品別。
